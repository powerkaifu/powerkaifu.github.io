CSS - 盒模型

發表於 2020-08-29

盒模型

CSS 在視覺的呈現是創意無限，就我所知，沒有一本書可以涵蓋介紹完它的屬性與效果間的組合，在這些組合技上，CSS 有時是很容易接近的朋友，但有時會鬧鬧脾氣，需要一段時間了解它。而本篇主題盒模型（Box-Model）的是其中一。

在 CSS 的世界裡，是由各式各樣的盒子組成，而這些盒子指的是網頁最基礎的 HTML 元素，也就是說，當我們建立一個 HTML 元素其實等於建立一個 CSS 盒子，一個矩形的盒子，它通常被稱為元素盒（element box）。每個元素盒的中心是內容區域（content area），也就是放文字、圖片的區域，內容區域的周圍還有內距（padding）、邊框（border）、外距（margin），由這些屬性來形成盒模型。

其中，這些屬性值得注意的是，它們都是選擇性的，內距（padding）的背景（顏色、重複圖片）預設是可以顯示在內距之內，它決定內容區域（content area）與邊框（border）之間的留白。外距（margin）它一定是透明，但可以顯示父元素的背景，也就是盒子與盒子之間的距離。內距無法使用負數，外距可以使用複數。邊框沒有指定顏色，會自動渲染內容區的前景色，也就是 color，邊框也是不能有負值。

基本盒

盒模型可以由兩個基本盒子說起，區塊盒（block-level box）、行內盒（inline box）。這兩個盒子型態是 CSS 世界的大眾居民，大部分元素都是這兩種型態（按鈕元素表示哭哭），最經典的範例各自代表為 div、span，當我們建立 div 元素等同建立一個區塊盒，建立 span 則是行內盒，概念如此。

區塊盒

區塊盒這個居民是 CSS 世界的地主，它們掌控了土地空間的霸權，所以當它們出現，正常流向由左至右的絕對空間（前、後換行），也就是整塊土地都會完全被它們佔有！區塊盒在正常流向裡會垂直堆疊。若有垂直外距有重疊時，則較大的外距會吃掉較小的外距。區塊盒的盒屬性相關設定都可以正常作用。

區塊盒例子：div、h1、p、ul、li、table 等等。

行內盒

行內盒是一般居民，它們是被大地主奴役的居民，它們只能擁有一小塊、一小塊的空間（寬、高隨著內容，無法設定寬、高）。這些行內居民必須一個一個由左至右乖乖排列著，脫離不了地主的控制。

除非…它們有勇氣採取革命，獨立定位自己，讓自己進化成區塊級，不過…一山還有一山高，再厲害七十二變的孫悟空永遠脫離不了如來佛五指山的相對位置？還是像變了心的女朋友回不去？這一切就要看造物主的設定囉。

行內盒例子：span、a、em 等等。

行內區塊盒

行內區塊盒（Inline-Block Box），它是介於居民與地主的角色，它的外在行為是行內居民，也就是會像居民乖乖排列，但內在行為卻是區塊地主，區塊盒的盒屬性相關設定它都可以正常作用。
那麼，誰才是 CSS 世界掌握最多資源的超級大地主？答案呼之欲出。

誰才是超級大地主？

超級大地主指的是初始容器區塊（initial containing block），body 只能算是大地主，隱藏版的超級大地主是根元素，也就是 body 的上層 html 元素，html 元素建立的是初始容器區塊，而初始容器區塊是由瀏覽器視埠（viewport）決定本身的尺寸。這也是為甚麼 body 的高度百分比無效化，原來它被 html 這個超級大地主控制著。

Margin、Padding、Border

  • 外距、內距若設定百分比（如果有特殊需求），會根據父元素的寬度來計算，即便是上下外距、內距的百分比也是如此，而不是高度喔。
  • 外距可以負數，內距不可以負數。
  • 外距重疊現象指兩區塊外距發生垂直時，大外距會吃掉小外距，而不是兩者相加；左右外距並不會發生重疊。
  • 內距預設會有背景圖渲染。

display

display 屬性可以替盒子進行轉型態的設定，

  • display:contents

作用在元素上時，display:contents 會將元素從頁面格式化移除，將自身的子元素移到自身所在的層級。
以下例子會將 ul 視覺格式化，就像從來不曾存在過一般，不只邊框消失，ul 周圍的內容的上下外距也一同消失。

Box-sizing

可以重新定義 Box 寬度的計算方式。時下流行將此屬性值設定為 border-box，也就是 padding、border 會被計算在 width 裡面，這樣在排版時，只要把焦點放在處理 margin。

Overflow

如果對盒子設定高度 height，內容如果超過高度，則會產生內容溢出，一般頁面元素的高度通常不需要設定，會隨著內容長高。

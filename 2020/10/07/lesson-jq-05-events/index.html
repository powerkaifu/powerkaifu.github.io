第五章 - 事件處理

發表於 2020-10-07

事件旅程

事件觸發對於網站來說是很重要的行為，沒有事件的處理網站就缺乏與使用者乏互動，而當頁面一個事件發生時會產生傳遞行為，每個層次上的 DOM 元素都有機會處理這個事件，以下面模型為例:

事件捕獲（event capturing）

舉例來說，如果點擊了頁面中的 a 錨點元素，那麼 div、span、a 全都應該得到回應這次點擊的機會。允許多個元素處理點擊事件的策略叫做 事件捕獲（event capturing）。
在事件捕獲的過程中，事件首先會交給最外層的元素，接著再交給更具體的元素。因此點擊事件首先會傳遞給 div、然後 span，最後是 a。

div => span => a

事件冒泡（event bubbling）

另一種相反的策略叫做 事件冒泡（event bubbling）。即當事件發生時，首先會發送給最具體的元素，在這個元素獲得回應機會之後，
事件會向上冒泡到更一般的元素。a 會首先處理事件，然後是 span，最後是 div。會有這兩個事件旅遊模型是因為瀏覽器大戰 Netscape(捕獲) 與 微軟 IE(冒泡)搞出來的事。
因此最終底定 DOM 標準應該使用這兩種策略，首先，事件要從一般到具體進行捕獲，然後，再透過冒泡傳出 DOM 樹的頂層。而事件處理常式可以註冊到這個過程中的任何一部分。

a => span => div

將事件設定成 事件捕獲 或 事件冒泡。

  • addEventListener 最後一個參數設為 true，事件捕獲，從外面開始。
  • addEventListener 最後一個參數設為 false（預設），事件冒泡，從裡面開始。
